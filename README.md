## 🛒 E-Commerce Manual Testing Portfolio

## 👩‍💻 About This Project

This is a **Manual Testing Portfolio Project** for an E-Commerce Web Application.
It covers end-to-end QA activities including test planning, test scenario creation,
detailed test case writing, and bug reporting — following real-world QA practices.

## 🧪 Testing Artifacts

### 📋 Test Plan
- Scope, objectives, and testing approach
- In-scope and out-of-scope features
- Test environment details
- Entry and exit criteria
- Risk analysis

### 📝 Test Scenarios
- 44 high-level test scenarios
- Covers all 9 functional modules
- Classified by Priority and Test Type

### ✅ Test Cases
- 50 detailed test cases
- Each case includes: Preconditions, Step-by-step actions, Expected Result, Actual Result, Priority, Status
- Linked to Test Scenarios via TS_ID

### 🐛 Bug Reports
- 25 bugs reported with full details
- Fields: Bug ID, Severity, Priority, Steps to Reproduce, Expected vs Actual Result, Environment, Status
- Includes Critical security bugs (SQL Injection, Auth bypass)
